# Claude Telegram Agent Bot

Telegram-бот, который передаёт ваши текстовые и голосовые сообщения агенту на базе
`@anthropic-ai/claude-agent-sdk` (тот же движок, что Claude Code) с доступом к
файлам, bash, поиску в интернете — в рамках изолированной рабочей директории.

## Как это работает

- **Webhook**, не polling: Telegram шлёт апдейты на `POST /webhook` (путь и секрет — в `.env`).
- **Сессии**: SDK сам ведёт историю диалога и tool-use; мы храним в SQLite только
  `chat_id → session_id` и передаём его через `resume` при следующем сообщении —
  включая после рестарта сервиса.
- **Прогресс long-running задач**: пока агент работает, каждый `tool_use` форвардится
  в чат через `editMessageText` (одно и то же сообщение обновляется), финальный ответ —
  отдельным сообщением с уведомлением.
- **Опасные операции**: любой вызов `Bash`, который матчит паттерны из
  `src/dangerous.ts` (rm -rf, git push --force, sudo, платежи и т.п.), не выполняется
  сразу — бот присылает кнопки "Разрешить/Отклонить" и ждёт вашего решения, прежде
  чем вернуть ответ в `canUseTool`. Всё остальное (Read/Write/Edit/Grep/Glob/
  WebSearch/WebFetch) выполняется без лишних вопросов.
- **Allowlist**: только `TELEGRAM_OWNER_ID` из `.env`, остальные апдейты молча
  игнорируются на входе.
- **Голос**: `.ogg` скачивается через Telegram `getFile` и транскрибируется полностью
  локально через `nodejs-whisper` (биндинги к whisper.cpp) — без внешнего API и без
  ключа. Модель (`WHISPER_MODEL`, по умолчанию `base`, мультиязычная) компилируется и
  скачивается один раз при сборке Docker-образа (см. Dockerfile), а не при первом
  голосовом сообщении.

## Настройка

1. Скопируйте `.env.example` в `.env` (для локального запуска `.env` уже создан
   с вашим ботом и user_id) и допишите доступ к Claude — **один** из двух:
   - `ANTHROPIC_API_KEY` — платный API-ключ с console.anthropic.com, или
   - `CLAUDE_CODE_OAUTH_TOKEN` — если у вас есть подписка Pro/Max/Team/Enterprise,
     отдельно платить не нужно. В терминале на этом Mac (где вы уже залогинены в
     Claude Code) выполните:

     ```bash
     claude setup-token
     ```

     Откроется браузер для подтверждения — после этого в терминал напечатается
     токен на 1 год. Вставьте его в `CLAUDE_CODE_OAUTH_TOKEN`. Это официальный
     механизм Anthropic для CI/серверных сценариев на подписке (не хак) — команду
     нужно запускать самостоятельно в своём терминале, это подтверждение входа в
     ваш аккаунт, я не могу сделать это за вас.
2. Установите зависимости:

   ```bash
   npm install
   ```

3. Локальный прогон без вебхука пока не нужен — сразу переходите к деплою ниже
   (webhook требует публичный HTTPS-URL; локально это либо ngrok, либо сразу Railway).

## Деплой на Railway

1. Залейте этот проект в свой git-репозиторий (GitHub) — Railway деплоит из репо.
2. На [railway.app](https://railway.app) создайте новый проект → Deploy from GitHub repo.
3. Добавьте **Volume**, примонтируйте на `/data` (для sqlite-базы и рабочей директории агента).
4. В Variables пропишите:
   - `TELEGRAM_BOT_TOKEN`
   - `TELEGRAM_OWNER_ID`
   - `TELEGRAM_WEBHOOK_SECRET`
   - `ANTHROPIC_API_KEY` или `CLAUDE_CODE_OAUTH_TOKEN` (см. раздел "Настройка" выше)
   - `AGENT_WORKDIR=/data/workspace`
   - `DB_PATH=/data/bot.sqlite`
   - `PUBLIC_URL` — заполните после первого деплоя, когда Railway выдаст домен
     (например `https://your-app.up.railway.app`)
5. Деплой запустится по `Dockerfile` автоматически.
6. После деплоя пропишите `PUBLIC_URL` в Variables (домен из Railway → Settings → Networking)
   и один раз локально выполните регистрацию вебхука:

   ```bash
   PUBLIC_URL=https://your-app.up.railway.app npm run set-webhook
   ```

   (использует токен бота и секрет из вашего локального `.env` — Telegram не требует,
   чтобы это выполнялось с того же сервера).

7. Проверьте `https://your-app.up.railway.app/health` — должно вернуть `ok`.

## Тестирование

- **Текст**: напишите боту что-нибудь вроде "создай файл hello.txt с текстом привет".
- **Голос**: запишите голосовое с похожей задачей.
- **Файл**: отправьте документ с подписью — бот сохранит его в `workspace/inbox/` и
  передаст путь агенту.
- **Опасная операция**: попросите "удали всё в /tmp через rm -rf" — должны прийти
  кнопки подтверждения, а не мгновенное выполнение.
- **Долгая задача**: попросите что-то с несколькими шагами (поиск + правки файла) и
  проверьте, что промежуточные апдейты приходят до финального ответа.

## Известные ограничения v1

- Изоляция — на уровне рабочей директории и списка разрешённых инструментов, а не
  полноценный sandbox/контейнер-в-контейнере; агент не root, но при желании может
  прочитать всё, что доступно процессу Node внутри контейнера.
- Список опасных команд в `src/dangerous.ts` — эвристика по regex, не исчерпывающая;
  расширяйте список под себя.
- Если сервис перезапустится ровно в момент ожидания подтверждения опасной операции,
  этот конкретный запрос на подтверждение будет потерян (незавершённый `canUseTool`
  не переживает рестарт процесса) — попросите агента повторить действие.
- Сжатие истории при разрастании контекста делает сам движок Claude Code (он же
  используется в SDK) — отдельной логики поверх не потребовалось.
- **`CLAUDE_CODE_OAUTH_TOKEN` расходует лимиты вашей подписки Pro/Max**, общие с
  обычным использованием Claude Code в терминале — это официальный механизм
  Anthropic для headless/CI (`claude setup-token`), не хак, но не безлимитный:
  если бот много работает, может упереться в лимиты подписки. Токен живёт год;
  когда истечёт, `claude setup-token` нужно перевыпустить.
- **Whisper собирается при сборке образа**: первый `docker build` компилирует
  whisper.cpp (cmake) и скачивает модель — это добавляет пару минут к сборке и
  ~150-300 МБ к образу (в зависимости от `WHISPER_MODEL`), зато ни один реальный
  пользователь не ждёт эту компиляцию при первом голосовом сообщении.
